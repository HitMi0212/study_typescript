타입스크립트의 장점

1. 에러의 사전 방지 2. 코드 가이드 및 자동완성(개발 생산성 향상) 타입스크립트 라이브러리 선택 1. 폴더 접근 2. npm i typescript -g 컴파일 tsc "파일명.ts" -> 번거로우니 웹팩을 사용하는걸 추천 타이핑 : 타입이 정의되지 않은 코드에 타입을 입혀주는 행위 // 타입 정의 type Todo_type = { id: number; title: string; done: boolean; }; // 인터페이스 interface Todo { id: number; title: string; done: boolean; }; 인터페이스와 타입 별칭의 차이점 - 확장 가능/불가능 - 인터페이스는 확장 가능 - 타입별칭은 불가능 유니온 타입 : 하나 이상의 타입 -> | 를 사용 타입 가드 : 특정 타입으로 타입의 범위를 좁혀나가는(필터링하는) 과정 유니온 타입의 특징 - 유니온 타입의 모든 속성을 접근할 수 있을 것 같지만 공통 속성만 접근 가능 인터섹션 타입 : 하나 이상의 타입을 모두 포함하는 타입 -> & 를 사용 프로토타입 var user = {name: 'user', age: 100}; var admin = {}; // user의 속성을 내려받아 사용할 수 있다.(상속) admin.__proto__ = user; 기존 타입 정의 방식과 제네릭의 차이점 - 기존 방식의 경우 다른 타입을 받기 위해 함수를 중복해서 작성해야한다. - 유니온 타입을 사용하는 경우 반환값의 타입 또한 유니온 타입으로 받게된다. -> 제네릭을 사용하면 타입별로 함수를 작성하지 않고 반환값의 타입 또한 정의할 수 있게된다. 타입 추론 - 변수를 선언하거나 초기화 할 때 타입이 추론된다. - 이외에도 변수, 속성, 인자의 기본 값, 함수의 반환 값 등을 설정할 때 타입 추론이 일어난다. Best Common Type - 추론하여 정의한 가장 근접한 타입 문맥상의 타이핑 - 문맥상의 타이핑(타입 결정)은 코드의 위치(문맥)를 기준으로 일어난다. 예시 코드 window.onmousedown = function(mouseEvent) { console.log(mouseEvent.button); //<- OK console.log(mouseEvent.kangaroo); //<- Error! }; 위 코드를 타입스크립트 검사기 관점에서 보면 window.onmousedown에 할당되는 함수의 타입을 추론하기 위해 window.onmousedown 타입을 검사합니다. 타입 검사가 끝나고 나면 함수의 타입이 마우스 이벤트와 연관이 있다고 추론 앞의 예제와 마찬가지로 오른쪽의 함수는 window.onscroll에 할당되었기 때문에 함수의 인자 uiEvent는 UIEvent으로 간주됩니다. 그래서 앞에서 봤던 MouseEvent와는 다르게 button 속성이 없다고 추론합니다. 그러므로 uiEvent.button에서 에러가 나죠. 여기서 만약 문맥상 타이핑을 좀 더 이해하고자 한다면 아래와 같이 코드를 바꿔볼 수도 있습니다. const handler = function(uiEvent) { console.log(uiEvent.button); //<- OK } 오른쪽 함수 표현식이 앞의 예제와 동일하지만 함수가 할당되는 변수만으로는 타입을 추정하기 어렵기 때문에 아무 에러가 나지 않습니다. 타입 호환 특정 타입이 다른 타입에 잘 맞는지를 의미한다. 구조적 타이핑 - 코드 구조 관점에서 타입이 서로 호환되는지 여부를 판단.
